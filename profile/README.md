## 搬瓦工的端口逻辑，和别人不一样

大多数 VPS 服务商，SSH 默认端口就是 22，买回来就是这个。但搬瓦工不是。

**搬瓦工的 SSH 端口是随机生成的**，每次重装系统之后，端口号会重新随机分配，通常是一个 10000 以上的五位数，比如 `28833` 或者 `51234` 之类的，总之不是 22。

这个设计本意是为了安全——22 端口是全球扫描最频繁的端口，换一个随机高位端口，能减少很多无效的暴力破解尝试。但也因此带来了麻烦：每次重装系统都要重新查一次端口；装了宝塔面板之后，宝塔只开放了 22 端口，搬瓦工的随机端口直接被防火墙拦住，连不上 SSH；还有人买了多台搬瓦工，记不住每台的端口号……

所以在折腾搬瓦工之前，**先搞清楚自己的端口号**，是第一步。

## 搬瓦工端口不通、连不上 SSH 的原因分析

遇到连不上 SSH 的情况，不要慌，先排查原因。通常分这几种：

**原因一：IP 被封**

最惨的情况。整个 IP 都被封了，从国内根本 ping 不通。需要通过 KiwiVM 面板检查 IP 状态，确认是否被封。被封之后要么付费换 IP（目前是 8 美元），要么迁移机房（迁移会获得新 IP，CN2 GIA-E 套餐支持免费迁移）。

**原因二：端口被封**

这个要好处理一些。IP 可以 ping 通，但 SSH 端口不通。换一个端口，重新连接就行。具体怎么判断和怎么换，后面详细说。

**原因三：服务没有监听端口**

有朋友刚装了一个服务，发现某个端口不通，就以为是被封了。其实不是——搬瓦工 VPS 本身不存在"防火墙全部开放所有端口"这种概念，一个端口能连通，前提是有服务在监听这个端口。比如 SSH 服务监听 SSH 端口，所以 SSH 端口通；你跑了个 web 服务在 8080，那 8080 才会通。端口没服务监听，显示"不通"是正常的。

**原因四：宝塔/面板防火墙拦截**

安装了宝塔面板之后，宝塔的安全功能会启用防火墙，默认只放行 22 端口。但搬瓦工的 SSH 端口不是 22，所以直接被拦住了。解决方法是进入宝塔面板的"安全"页面，手动添加放行规则，允许你的 SSH 端口通过。

## 如何检测搬瓦工端口是否被封

确认了 IP 没问题，但还是连不上 SSH，接下来就要测试端口是否被封。

**方法：使用 port.ping.pe 在线检测工具**

1. 打开网址：`http://port.ping.pe`
2. 在输入框中填入你的 IP 和端口，格式为 `IP:端口`，例如 `1.2.3.4:28833`
3. 注意冒号是**英文冒号**
4. 点击检测，等待结果

结果分析：

| 检测结果 | 含义 |
|----------|------|
| 境外正常 + 境内正常 | 端口完全正常，连不上另有原因 |
| 境外正常 + 境内全部不通 | 端口或 IP 被封，国内无法访问 |
| 境外全部不通 + 境内全部不通 | 端口未正确开启，不是被封，检查服务是否在运行 |
| 境内个别不通 | 部分地区网络问题，端口本身正常 |

如果结果是"境外正常、境内全部不通"，那么基本可以确认端口被封了，需要更换端口。

## 搬瓦工修改 SSH 端口完整步骤

端口被封、或者想把随机端口改成好记的端口，都可以按下面的步骤操作。

因为 SSH 端口已经不通，所以**不能通过 SSH 登录来修改端口**。这时候要用 KiwiVM 面板提供的在线终端（Web Terminal）来绕过这个限制。

**第一步：进入 KiwiVM 在线终端**

1. 登录 KiwiVM 面板
2. 在左侧菜单找到 **Root shell – interactive**（或 Root shell – advanced）
3. 点击 **Launch**，弹出基于 Web 的终端窗口
4. 在终端里用 root 用户名和密码登录（输密码时不显示任何字符，正常现象）

**第二步：打开 SSH 配置文件**

输入以下命令：

bash
vim /etc/ssh/sshd_config


如果提示 vim 不存在，先安装：

- Ubuntu / Debian：`apt install -y vim`
- CentOS：`yum -y install vim`

**第三步：修改端口号**

打开配置文件后，找到带有 `Port` 的行，大概长这样：


Port 28833


按 `i` 键进入编辑模式，把端口号修改成你想要的值。建议选一个 10000～65535 之间的数字，避免和常用服务端口冲突。

修改完成后：
- 按 `Esc` 退出编辑模式
- 输入 `:wq` 保存并退出

**第四步：重启 SSH 服务**

bash
/etc/init.d/sshd restart


或者（某些系统）：

bash
systemctl restart sshd


**第五步：用新端口连接**

回到 SSH 客户端，用新的端口号重新连接测试。如果能连上，修改成功。

如果无论如何都连不上，还可以回到 KiwiVM 的 Root shell 再次进入系统检查配置是否正确。

## 搬瓦工端口修改后的注意事项

修改完端口之后，有几件事别忘了：

**更新 KiwiVM 显示的端口**：有时候 KiwiVM 面板显示的端口和实际系统端口不一致，这是因为面板显示的是初始端口，不会随着你手动修改而自动更新。以实际能连上的端口为准。

**如果装了宝塔面板**：必须进入宝塔面板 → 安全页面 → 添加放行规则，把新端口加进去，否则宝塔的防火墙会继续拦截。

**如果是 firewalld（CentOS 7+）**：需要额外执行放行命令：

bash
firewall-cmd --permanent --add-port=新端口号/tcp
firewall-cmd --reload


**记录新端口**：改完之后找个地方记下来，下次连接别再找不到了。

## 搬瓦工端口相关的常见问题

**Q：重装系统后端口变了，怎么找新端口？**

回到 KiwiVM 面板主页查看，重装系统后面板会同步显示最新的 SSH 端口。

**Q：我想把端口改回 22 可以吗？**

技术上可以，修改 `sshd_config` 里的端口号改成 22 即可。但从安全角度来说不建议，22 是最常被扫描的端口，暴力破解攻击很频繁。用一个高位随机端口会安全很多。

**Q：端口被封了，换新端口之后还会被封吗？**

换一个端口之后，新端口理论上是干净的，不会立即被封。但如果继续跑的服务触发了被封的原因，新端口还是可能被封。建议换端口之后排查一下是否有违规使用的情况。

**Q：搬瓦工 VPS 是不是自带防火墙，需要手动开放端口？**

搬瓦工 VPS 本身**默认不开启防火墙**。如果你的系统是 Ubuntu/Debian，通常没有防火墙，端口只要有服务监听就是通的。如果是 CentOS 系统，可能有 iptables 或 firewalld，需要手动放行。装了宝塔面板的话，宝塔自带防火墙功能，需要在宝塔内放行端口。

**Q：80 和 443 端口通吗？**

只要你在 VPS 上部署了对应的服务（比如 Nginx、Apache 等跑在 80/443 端口），这两个端口就会通。没有服务监听，自然是不通的——这是正常现象，不是被封。
